Technical Reference : Database Administration : GEN 1 : Backup : Method 1

Recommended Backup Matrix #1 (primary)

This is the recommended backup procedure, and is structured to aid in the successful retention of all data for a one-year period. NOTE that the daily log databases are different from the summary database. The daily logs are dynamically created during overnight processing, and are not specified during system installation. They must be backed up manually during each backup period. Also note that 2 tapes are used for each backup SET. This is for redundancy as each tape within a set stores a number of consecutive daily logs. Each backup SET should last for the predetermined amount of time. For example, a 50 bus fleet would be able to use one SET of daily backup tapes for a full 6 months. At the end of 6 months, the first SET is put aside in a secure location, and the second SET begins to be used. Each tape within a Daily Log Database backup SET will mirror the other.

Backup of Daily Log Databases (this is a manual operation to be performed daily):


      Daily Log File backup – Placing redundant data on 2 tapes each day. These same 2 tapes are reused for the length of the SET. The following is an example of the procedure that would be used on a fleet with 50 buses:

      Due to the fleet size of 50 buses, it has been determined from data flow that each SET of Daily Log backup tapes will last for 6 months. Each day it would be necessary to place tape 1 of "SET 1" into the tape device and run a backup on the latest daily information. This step is repeated for tape 2 of "SET 1" after tape 1 has completed the backup procedure. When or more daily logs reside on the system, it is necessary to delete all logs over 10 days old after backup. Thus after 5 days of backing up from 01/01/98, both tapes 1 and 2 of "SET 1" would have the following databases:

      Log19980101
      Log19980102
      Log19980103
      Log19980104
      Log19980105

      Set 1 = 2 mirrored tapes for January thru June
      Set 2 = 2 mirrored tapes for July thru December

      Each tape will last for the time frame specified at the end of this document. The recommended frequency of backing up Daily Log databases is listed below.

    1. Saturday – backup on Sunday morning with tapes 1 and 2
    2. Sunday – backup on Monday with tapes 1 and 2
    3. Monday – backup on Tuesday with tapes 1 and 2
    4. Tuesday – backup on Wednesday with tapes 1 and 2
    5. Wednesday – backup on Thursday with tapes 1 and 2
    6. Thursday – backup on Friday with tapes 1 and 2
    7. Friday – backup on Saturday with tapes 1 and 2

Backup of Summary databases: ().

This operation is to be scheduled by the system administrator, actual database names may vary.  The execution of this procedure relies upon the replacement of tapes each day according to the schedule listed below. December will not have a monthly tape due to the fact that each tape is essentially a full backup of the data that has accrued rather than an incremental backup from the previous date; therefore, December 31st would utilize the year end tapes. Two tapes are recommended for the year end backup for redundancy.

Daily use – 7 tapes

Yearend use – 2 tapes

Backup Procedure – placing data on 1 tape each day. The tapes should be used / reused as follows:

    1. Saturday – Reuse each Saturday
    2. Sunday – Reuse each Sunday
    3. Monday – Reuse each Monday
    4. Tuesday – Reuse each Tuesday
    5. Wednesday – Reuse each Wednesday
    6. Thursday – Reuse each Thursday
    7. Friday – Reuse each Friday
    8. Year End Tape 1 – Use on December 31
    9. Year End Tape 2 – Use on December 31
