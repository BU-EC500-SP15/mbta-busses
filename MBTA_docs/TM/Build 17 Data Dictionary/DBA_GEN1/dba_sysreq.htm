The SQL Server should reside on a Member Server. It is possible to have this machine also be the Backup Domain Controller, though the amount of network traffic involved does not make this the ideal situation. It is understood that individual properties will have procurement limits, and that machines will have to be budgeted, but if at all possible, the above mentioned scenario is preferred.

The following pages outline the recommended backup procedure to be followed. The total number of disks and their replacement frequency has been listed at the end of each of the two recommended procedures. The pages following the backup procedures outline the accepted restoration of data back into the SQL Database. In addition to the above mentioned size specifications, if it is financially feasible to do so we recommend the use of a RAID disk subsystem (Redundant Array of Independent Disks).

All examples within this document are based upon a fleet size of 50 buses. The numbers have been extrapolated to included up to 200 buses, and can expand further as needs arise.
