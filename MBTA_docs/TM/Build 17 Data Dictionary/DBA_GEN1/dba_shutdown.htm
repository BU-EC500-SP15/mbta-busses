System Shutdown Procedures

It is recommended that the system undergo periodic shutdown in order to clean out virtual memory and physical RAM. The recommended order of shutdown is listed below, as well as possible problems that may occur. 

  1. Shut down \ Power down Backup RNC
  2. Shut down \ Power down Primary RNC
  3. Shut down \ Power down all Dispatch Machines
  4. Shut down \ Power down TransitMaster Server
  5. Shut down \ Power down SQL Server
  6. Shut down \ Power down Backup Domain Controller (if separate machine)
  7. Shut down \ Power down Primary Domain Controller (if separate machine)
  8. Wait for 60 seconds
  9. Start Primary Domain Controller (if separate machine)
  10. Start Backup Domain Controller (if separate machine)
  11. Start SQL Server
  12. Start TransitMaster Server
  13. Start Primary RNC
  14. Start Backup RNC
  15. Start Dispatch Machines
    16.

It is recommended that the above procedure be implemented at least once per week. Each individual property will determine which day of the week will provide for the most optimum shut down time. The start time for the complete system shutdown should precede the implemented backup times. For example, if the backup is scheduled to initiate at midnight, then the shutdown should be executed no later than 11:30 pm.

The procedure listed above will automatically perform all Service shutdowns and restarts. If a Service fails to start during boot time, examine the Event Viewer to determine which service did not start. If any TransitMaster services did not start, examine the Control Panel / Services window to see that all TransitMaster services are listed as Automatic, and that they have the appropriate startup logon.

Should an error occur while bringing the RNC machines back online, it may be necessary to reboot the modems. It is recommended that the reboot of BOTH modems (one on the RNC and one at the transit center) should occur simultaneously. This can be achieved via a telephone switch box located at the remote site. If further problems persist, please contact the TransitMaster tech support representative for the respective property.
