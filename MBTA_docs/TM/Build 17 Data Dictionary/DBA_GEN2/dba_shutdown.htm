Database Administration : GEN 2 : System Shutdown

The system should be shutdown at least once per week to refresh virtual memory and physical RAM. Follow these steps to complete this procedure:

  1. Shut down \ Power down Backup RNC
  2. Shut down \ Power down Primary RNC
  3. Shut down \ Power down all Dispatch Machines
  4. Shut down \ Power down TransitMaster Server
  5. Shut down \ Power down SQL Server
  6. Shut down \ Power down Backup Domain Controller (if separate machine)
  7. Shut down \ Power down Primary Domain Controller (if separate machine)
  8. Wait for 60 seconds
  9. Start Primary Domain Controller (if separate machine)
  10. Start Backup Domain Controller (if separate machine)
  11. Start SQL Server
  12. Start TransitMaster Server
  13. Start Primary RNC
  14. Start Backup RNC
  15. Start Dispatch Machines

Each property will determine the optimal day and time to perform this shutdown and restart of the system. The start time for the complete system shutdown should precede the scheduled backup.  Allow 30 minutes for the shutdown and restart to complete before the backup is scheduled to begin.

The procedure listed above will automatically perform all Service shutdowns and restarts. If a Service fails to start during boot time, examine the Event Viewer to determine which service did not start. If any TransitMaster services did not start, examine the Control Panel / Services window to see that all TransitMaster services are listed as Automatic, and that they have the appropriate startup logon.

If an error occurs while bringing the RNC machines online, it may be necessary to reboot the modems. The reboot of BOTH modems (one on the RNC and one at the transit center) should occur simultaneously. This can be achieved via a telephone switch box located at the remote site. If further problems persist, please contact the TransitMaster tech support representative for the respective property.
