Technical Reference : Reporting

TransitMaster Reporting provides numerous prepared reports as well as the capability to build custom reports.  Data Flow Diagram.

Prepared Reports

Driver Activity Summary of communications, violations, logons, and schedule adherence by driver.
Route Activity Summary of communications, violations, logons, and schedule adherence by route.
Driver Text Message Activity Echo of text messages transmitted to and from a driver.
Driver Performance  Count of time point crossings and corresponding schedule adherence by driver.
Route Block Performance Count of time point crossings and corresponding schedule adherence by route block.
Route Performance  Count of time point crossings and corresponding schedule adherence by route.
Vehicle Performance   Count of time point crossings and corresponding schedule adherence by vehicle.
Maintenance Warnings by Driver Summary of mechanical alerts by driver.
Maintenance Warnings by Vehicle  Summary of mechanical alerts by vehicle.
MDT Software Version  Record of installed components upon MDT startup.
Passenger Count Messages  Record of passengers on/off at each stop.
Passenger Counts by Trip Summary of passengers on/off by vehicle, trip serial #, and time of day.
Pull-Out Activity Detail for each block - scheduled and actual vehicle, scheduled and actual pull-out time.
Pull-In Activity Detail of each block - scheduled and actual vehicle, scheduled and actual pull-in time.
Route Deviations Log of driver, vehicle, geo node, direction, and timestamp for each route deviation message.
Schedule Deviations Log of driver, vehicle, timestamp, and deviation minutes for each time point.
Schedule Adherence Log of time point crossings by route and block with driver, actual time crossed, and adherence.
Vehicle Miles and Hours Summary of revenue miles, non-revenue miles, revenue hours, and non-revenue hours by calendar day.

The following stored procedures are located in the TMreporting database and are run for each service day to summarize, or condense data from TMdailylog:

p_rptg_load_communication_sum
p_rptg_load_daily_message_text
p_rptg_load_driver_logon
p_rptg_load_driver_violation
p_rptg_load_mdt_soft_version
p_rptg_load_mechanical_warning
p_rptg_load_pass_count
p_rptg_load_pullinout_summary
p_rptg_load_route_deviation
p_rptg_load_sched_adherence
p_rptg_load_sched_deviation
p_rptg_load_veh_miles_hrs

All procedures have 1 call parameter - the calendar_id for the date to be summarized.  These procedures can be run in any order or combination.   Any individual procedure can be re-run for a given calendar day.  All summary detail for the specified day (calendar_id) is deleted and reloaded.

The TMreporting..Report_Control table must be populated with one row to identify the transit division for reporting.
The TMreporting..Summary_Control table identifies the date and time that each reporting table was loaded with daily log information for each calendar day (calendar_id).
