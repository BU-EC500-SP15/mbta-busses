Technical Reference : Database Administration : GEN 2 : SQL Server Agent, Operators, and Alerts

SQL Server Agent and Email Notifies

The SQL Server Agent can send an email message to a database administrator when a configured event occurs.  To enable the Agent to send an email, you must identify the mail server account where the messages will originate.  To set this up in Enterprise Manager, right click on "SQL Server Agent" and select "Properties".  On the General tab of the SQL Server Agent Properties window, identify the NT account that the SQL Server Agent should use when the service is started, and specify an email user account for sending messages.  [ Show Me ]

Operators

SQL Server provides an Alert facility to notify a database administrator when a configured event has occurred.  Each database administrator that is to receive an email or page is set up as an Operator.  To add an Operator to the database server using Enterprise Manager, select Management, then select SQL Server Agent, and then select Operators.  A list of operators will appear in the panel to the right.  Right-click on "Operators" and select "New Operator...". [ Show Me ]

Alerts

Database server errors can be trapped by the SQL Server Agent.   For each alert, or event, a response is defined.  To add an Alert to the database server using Enterprise Manager, select Management, then select SQL Server Agent, and then select Alerts.  A list of alerts will appear in the panel to the right.   Right-click on  "Alerts" and select "New Alert...".   [ Show Me ]

There are two tabs in the Alert Properties window.  In the General tab, identify the type of message that should trigger the alert.  In the Response tab, identify the response the Agent should take when the alert is triggered.  You may choose to email or page an Operator and/or run a script.
